What is United Edit Suite?

United Edit Suite (UES) is a companion tool for the United inventory platform. It automatically detects the connected United version and loads the active customer‚Äôs data. The default view lists areas that have been counted and not marked as printed; items already printed disappear from this list, and items not yet counted don‚Äôt appear. Filters let you include printed items or show not-yet-counted areas.

Menu depth rule: Keep United open but do not be deeper than one level inside submenus while using UES. Being in the final file or >1 level deep can lock files and trigger errors until you back out. Use UES freely from high-level menus like Inventory Proofing.

Use the Refresh button whenever changes occur in United (new uploads, items marked as printed). Refresh syncs those changes into UES.

Mark-as-Printed behavior: UES recognizes items marked as printed in United, but United does not recognize items marked as printed inside UES. To avoid confusion, pick one place to mark items as printed and stay consistent.

Reports (Financial-style)

Cost Totals

  • If percentages are given in United, you will see a banner at the top of the report window telling you that you are viewing cost adjust totals.

Split Window

  1. Open Edit on a location, choose employee (if multiple).
  2. Pick the exact boundary record (by SKU or price point, e.g., $4.49). UES previews remaining totals for current vs. new location/area.
  3. Use Search Record if multiple matches exist; choose field (price, SKU, etc.), then select the unique match to complete the split.

Category change & manual entry (in-report edit)

  • Category Change‚Ä¶: Choose any category present in the location, pick a new category, and enter the amount to shift (e.g., move $500 from 14 ‚Üí 3). UES applies the adjustment and updates totals.
  • Manual Entry‚Ä¶: Like United‚Äôs manual entry but prefilled with the chosen location. Enter category + total repeatedly for fast backroom adds; close to commit all entries.
  • Delete Location‚Ä¶: Delete entire location, only a specific upload in the location, or just one employee‚Äôs portion.

Zero-total locator

If a location shows $0 and you‚Äôre unsure where its counts went: click the location number. Because total is zero, UES opens a helper that lets you pick who counted it, shows all their locations with totals (including merged cases), and suggests five likely matches based on proximity. Details window

Details is similar to United‚Äôs final file view. By default it‚Äôs read-only; click Unlock to enable edits for supported fields.

  • Global search across category, price point, SKU, or description.
  • Right-click a cell to filter to that value (e.g., show all for a SKU).

Editing

  • Edit location and category on a single record.
  • Drag to multi-select rows, then Quick Change‚Ä¶ to update all (or Delete).
  • Add Record creates a blank line (file name = Edit Suite).
  • When you edit a SKU, UES validates against the live customer master; price/description (and hidden fields like cost) update automatically.

MassEdit

  • Select a beginning record then click MassEdit. Fields auto-populate.
  • Clear a field to apply to all (no need to type ‚ÄúALL‚Äù).
  • Leaving end record blank extends to the last record UES finds within the parameters (e.g., last record for upload 13 in area 40041, location 04-010).
  • Use Save Changes; if you try to exit with unsaved edits, UES prompts you to save.

View Details from Reports shows only the selected location. View Details (All) shows details across all currently viewed areas. In Scan Reports, View Details (Final File) opens the entire final file.

Audits (Desktop & Tablet)

Audits support sorting on all fields, show grand totals, and capture full change history.

Preparing & printing

  • Search by name, area, or location. Toggle options (e.g., hide employee names on printed audits).
  • Select audits ‚Üí Print Audits to generate PDFs. UES remembers what‚Äôs been printed even after closing. Reprint specific audits by using Re-Enable.
  • Export Audits mirrors printing: exports only the enabled set.

Tablet workflow (same network)

  1. Click Get Address to obtain the device URL (IP + port) and open it on a tablet.
  2. On tablet, view audits by Name or by Location. Review includes: add record, mark for deletion, change SKU (invalid SKUs highlight red), edit quantity; for financial inventories, price and category can be edited too.
  3. When a location is finished, mark it Complete. Completed names/locations disable or disappear accordingly.

Importing & resolving audits

  1. On desktop, click Import Audits to pull tablet changes.
  2. Review the feed: added records (SKU/price + units), deletions, SKU changes (before/after, including price changes), unit changes, and not-found SKUs.
  3. Use Accept All, Reject individual items, or leave some pending to revisit later. Pending items remain visible until resolved.

Daily backups of audits (PDF + logs)

  • Use Back Up Exports at day‚Äôs end to select a folder. UES creates an /audits subfolder with a PDF per location (green = accepted, red = rejected, no color = unchanged).
  • Each import produces a text log capturing what changed, when the import occurred, and delta amounts per record.

Recovery Report (MODAS ‚ÄúMayday‚Äù)

Use Recovery Report to create an audit per employee with the 10 most recent items from the final file. This helps teams quickly resume work in a Mayday scenario.

Enable / Disable / Re-Enable

  • Enable All: Enable audits for all current rows so they can be printed or exported.
  • Disable All: Clear all enables.
  • Re-Enable All: Re-enable items that were previously printed/exported and are currently enabled, for reprint/re-export.

Options & Automation

  • Auto-update Areas: For each incoming location, if its current area differs from the most recent prior‚Äôs area, UES updates the current to match the prior (fixes forgotten area changes).
  • Auto-fix increments of locations: If a location ends with 1 or 6 and didn‚Äôt previously exist, UES adjusts it to end with 0 or 5.
  • View Production: Displays current production metrics for all employees.
  • View Prior Inventory: Allows the user to select the last inventory backup and view the final file of that inventory in the details window.

üß± Build Date/Time Field

Purpose:
The Build Date/Time Field option allows you to manually populate the other5 field with formatted date and time values based on each record‚Äôs existing date and time fields.

How It Works

  1. Choose Options ‚Üí Build Date/Time Field.
  2. Select a desired format in the Date/Time Format dialog.
  3. The system then updates final.dbf, filling other5 for each record using the chosen format (e.g., mm/dd/yy hh:mm ‚Üí 10/10/25 14:45).
  4. If a record is missing either date or time, only the available part is written; missing components are left blank.

Notes

  • This process runs once per command; it is not automatic.
  • Existing other5 values are replaced during this operation.
  • Execution is batched for efficiency ‚Äî performance impact is minimal even for large final.dbf files.

üïì Auto Apply Date/Time

Purpose:
Enables automatic date/time stamping of new records as they are added to final.dbf, using the same formatting rules as Build Date/Time Field.

How It Works

  1. Enable Options ‚Üí Auto Apply Date/Time (checkable menu item).
  2. When enabled for the first time, the Date/Time Format dialog appears so you can select your preferred date/time layout.
  3. On each refresh or incremental update:
    • Any record where other5 is empty is filled with the current local date/time in the chosen format.
    • Records that already contain a value in other5 are skipped.
  4. Disable at any time by unchecking Auto Apply Date/Time.

Example Output

10/10/25 14:45
10/10/25 15:22
10/10/25 16:08

Performance Notes

  • Only affects rows where other5 is blank.
  • Uses a single batch update rather than per-row writes, ensuring fast execution even as final.dbf grows throughout the day.

Together, these two features allow either manual or automatic control over how and when timestamps are applied to your data, enabling both end-of-day and continuous processing workflows.

  • Emergency Backups (United): Every time you click Refresh and UES detects a changed final file, it creates a United backup in the designated folder.
  • Restore Customer Files: UES creates a folder when a new customer is detected on open. If you accidentally switch customers mid-inventory and lose printed reports/audits, restore the previous store to continue.
  • MySQL Backups (MobTrans): Lists historical MySQL backups (not United backups). You can open versions to inspect, then Import New MySQL and select the known-good file to restore.
