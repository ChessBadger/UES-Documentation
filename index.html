What is United Edit Suite?

United Edit Suite (UES) is a companion tool for the United inventory platform. It automatically detects the connected United version and loads the active customer’s data. The default view lists areas that have been counted and not marked as printed; items already printed disappear from this list, and items not yet counted don’t appear. Filters let you include printed items or show not-yet-counted areas.

Menu depth rule: Keep United open but do not be deeper than one level inside submenus while using UES. Being in the final file or >1 level deep can lock files and trigger errors until you back out. Use UES freely from high-level menus like Inventory Proofing.

Use the Refresh button whenever changes occur in United (new uploads, items marked as printed). Refresh syncs those changes into UES.

Main window layout

  • Managers/Auditors contains Manager Reports and Audits.
  • Data contains View Details and View All Uploads.
  • Reference Views contains View Production, View Prior Inventory, and View Emergency Backups.
  • Finalize contains Print Final Reports, Create Backup, and Print Cost Report when available.
  • Prep Tools contains Print Worksheet and Create Cards.
Mark-as-Printed behavior: UES recognizes items marked as printed in United, but United does not recognize items marked as printed inside UES. To avoid confusion, pick one place to mark items as printed and stay consistent.

Create Cards

Open Create Cards under Prep Tools and choose your customer under DC Setup to create DC5 cards for the current inventory.

Worksheets

Print Worksheets

Open Print Worksheet under Prep Tools to generate worksheet PDFs. You can choose to show no categories, categories, or category groups.

Highlights

Enable Highlights to visually alternate rows for locations with multiple categories so nothing is missed during.

Threshold

Set a Threshold to limit highlighting to locations with more than one category but no more than N categories. Example: Threshold = 4 highlights locations that have no more than 4 categories.

Category Cost Report

  • When all areas marked as printed are selected in the main window, the Print Cost Report button appears in Finalize.
  • This will generate a category report displaying the retail, cost, variance, and percentage chosen based on the values in United.
  • All values are converted using the values in the "Cost Pct" field of each category in United

Selecting Areas

  • Click to select individual areas; click–drag for multi-select.
  • With areas selected, choose Open Reports or Open Scan Reports.
  • Filters at the top of the list include Printed, Areas Not Counted, and View Recounts.

Finalize

Use Finalize on the right side of the main window for end-of-day reporting and backup tasks.

  • Print Final Reports opens a picker where you can choose areas, choose category detail level, and generate the final report set.
  • Create Backup lets you choose the United backup zip name and location, then also drops fresh Audits and Manager Reports folders into that same location.
  • Print Cost Report appears here when it is available for the current selection.

Reports (Financial-style)

The View Reports window mirrors United’s printed report style and supports in-place editing with helpful tooltips.

Cost Totals

  • If percentages are given in United, you will see a banner at the top of the report window telling you that you are viewing cost adjust totals.
  • The cost percentages are displayed next to each location number when applicable.
  • All totals are adjusted in the final file with negative entries upon opening the report window based on the current retail total of the location and the value in United.
  • All adjustments will have a file name of COST_ADJ in the final file.

Visibility controls

  • Toggle Categories and Category Groups to adjust report granularity.
  • When viewing by groups, clicking a group number toggles (hides/shows) underlying categories.

Bulk actions

  • Move area: Click an area number to move all its locations to a new area number.
  • Rename area: Click an area description to edit or annotate.
  • Relabel location: Click a location number to renumber. You can target one or multiple employees or All.
  • Edit location description: Click description to change text or add notes. Right-click → Mark as Gone prefixes “Gone ” to the name.
  • New Location…: Add a location with description/notes. This writes to United Location Setup and seeds a prior of $0 so the location appears on reports.
  • Quick Edit…: Quickly change or swap area/location numbers (e.g., swap 03090↔03100, set multiple to 03070).
  • Move category: Click a category number to move it. If grouped, pick a category with data.

    • When a location’s current description differs from a known prior description, the report shows a red ** prefix before the current text, and (if space allows) the prior description appears in red italics inside parentheses.

Print Report (PDF totals)

Use Print Report at the top to generate a PDF of the totals shown in the current view. The PDF mirrors what you see on screen and can be saved or printed.

Export to Manager Reports

  • Export Grouped writes the currently displayed areas as one shared export group.
  • Export Separate writes the same visible areas as separate exports with no shared grouping.

Visual indicators

  • 🧑‍🤝‍🧑 Multiple employees counted this location.
  • ⚠️ Location total is $0 but there were priors.
  • Red category rows = previously counted here, but missing in current inventory.

Recount indicators

  • Locations already marked for recount are shaded and labeled RECOUNT in the report window.
  • Recount state is preserved in the Manager Reports export so supervisors can continue tracking active recounts.
  • To clear te recount state, re-export the area. If no change has been made, that will be logged for the mamanger automatically.

Split Window

  1. Open Edit on a location, choose employee (if multiple).
  2. Pick the exact boundary record (by SKU or price point, e.g., $4.49). UES previews remaining totals for current vs. new location/area.
  3. Use Search Record if multiple matches exist; choose field (price, SKU, etc.), then select the unique match to complete the split.

Category change & manual entry (in-report edit)

  • Category Change…: Choose any category present in the location, pick a new category, and enter the amount to shift (e.g., move $500 from 14 → 3). UES applies the adjustment and updates totals.
  • Manual Entry…: Like United’s manual entry but prefilled with the chosen location. Enter category + total repeatedly for fast backroom adds; close to commit all entries.
  • Delete Location…: Delete entire location, only a specific upload in the location, or just one employee’s portion.

Zero-total locator

If a location shows $0 and you’re unsure where its counts went: click the location number. Because total is zero, UES opens a helper that lets you pick who counted it, shows all their locations with totals (including merged cases), and suggests five likely matches based on proximity. Double-click a candidate, verify in Details, then Update Location to move the data.

Scan Reports (Pieces/Units)

Identical functionality to Reports, but the layout is optimized for pieces/units style rather than financial three-variance format.

Manager Reports Window

Open Manager Reports from the main window to review the JSON report exports in one place. It is the supervisor view for area review status, active recounts, field questions, chat, and tablet connection status.

Top controls

  • Refresh reloads the exported report files and updates the summary cards.
  • Backup creates a zip backup that includes Audits plus the current Manager Reports export files.
  • Chat Log opens the shared chat history and clears the unread chat badge after viewing.
  • The Tablet status card shows whether the tablet is connected to the network or not.

Areas tab

  • Shows one row per area with area number, description, review status, review time, recount count, and question count.
  • Use Search and the Status filter to narrow the list to reviewed areas, items still to review, or areas with recount/question activity.
  • Selecting an area updates the details panel with location count, area totals, and rolled-up category totals.
  • Right-click an area and choose Remove Area to clear that export from the Manager Reports view.

Recounts and Questions

  • The Recounts tab lists active recount requests by area and location, including the reason and timestamp.
  • The Questions tab lists field questions and any reply already saved.
  • Selecting a recount or question updates the details panel with that location's current/prior totals and category breakdown.
  • When a question is selected, use the reply editor to save a response back into the report file.

Unread indicators

Red notification badges mark new recounts, questions, or chat messages that have not been viewed yet. The main window mirrors this unread state on the Manager Reports button.

Details window

Details is similar to United’s final file view. By default it’s read-only; click Unlock to enable edits for supported fields.

Search & filter

  • Global search across category, price point, SKU, or description.
  • Right-click a cell to filter to that value (e.g., show all for a SKU).

Editing

  • Edit location and category on a single record.
  • Drag to multi-select rows, then Quick Change… to update all (or Delete).
  • Add Record creates a blank line (file name = Edit Suite).
  • When you edit a SKU, UES validates against the live customer master; price/description (and hidden fields like cost) update automatically.

MassEdit

  • Select a beginning record then click MassEdit. Fields auto-populate.
  • Clear a field to apply to all (no need to type “ALL”).
  • Leaving end record blank extends to the last record UES finds within the parameters (e.g., last record for upload 13 in area 40041, location 04-010).
  • Use Save Changes; if you try to exit with unsaved edits, UES prompts you to save.

View Details from Reports shows only the selected location. View Details (All) shows details across all currently viewed areas. In Scan Reports, View Details (Final File) opens the entire final file.

All Uploads

  • Wintakes-like view of every upload: who counted it, dollar and unit totals. Values reflect the final file (i.e., after adjustments).
  • Delete Upload to remove a selected upload.
  • View Locations within an upload, or open Details filtered to just that upload.
  • Add employee number Right click to assign or change an employee for an upload.
  • Right click any upload to add or change the employee number in of that upload

Audits (Desktop & Tablet)

Audits support sorting on all fields, show grand totals, and capture full change history.

Preparing & printing

  • Search by name, area, or location. Toggle options (e.g., hide employee names on printed audits).
  • Select audits → Print Audits to generate PDFs. UES remembers what’s been printed even after closing. Reprint specific audits by using Re-Enable.
  • Export Audits mirrors printing: exports only the enabled set.

Tablet workflow (same network)

  1. Click Get Address Will provide a QR code that can be scanned via the camera app on the tablet.
  2. On tablet, view audits by Name or by Location. Review includes: add record, mark for deletion, change SKU (invalid SKUs highlight red), edit quantity; for financial inventories, price and category can be edited too.
  3. When a location is finished, mark it Complete. Completed names/locations disable or disappear accordingly.

Importing & resolving audits

  1. On desktop, click Import Audits to pull tablet changes.
  2. Review the feed: added records (SKU/price + units), deletions, SKU changes (before/after, including price changes), unit changes, and not-found SKUs.
  3. Use Accept All, Reject individual items, or leave some pending to revisit later. Pending items remain visible until resolved.

Daily backups of audits (PDF + logs)

  • Use Back Up Exports at day’s end to select a folder. UES creates an /audits subfolder with a PDF per location (green = accepted, red = rejected, no color = unchanged).
  • Each import produces a text log capturing what changed, when the import occurred, and delta amounts per record.

Recovery Report (MODAS “Mayday”)

Use Recovery Report to create an audit per employee with the 10 most recent items from the final file. This helps teams quickly resume work in a Mayday scenario.

Enable / Disable / Re-Enable

  • Enable All: Enable audits for all current rows so they can be printed or exported.
  • Disable All: Clear all enables.
  • Re-Enable All: Re-enable items that were previously printed/exported and are currently enabled, for reprint/re-export.

Options & Automation

  • Auto-update Areas: For each incoming location, if its current area differs from the most recent prior’s area, UES updates the current to match the prior (fixes forgotten area changes).
  • Auto-fix increments of locations: If a location ends with 1 or 6 and didn’t previously exist, UES adjusts it to end with 0 or 5.
  • View Production: Displays current production metrics for all employees.
  • View Prior Inventory: Allows the user to select the last inventory backup and view the final file of that inventory in the details window.

🧱 Build Date/Time Field

Purpose:
The Build Date/Time Field option allows you to manually populate the other5 field with formatted date and time values based on each record’s existing date and time fields.

How It Works

  1. Choose Options → Build Date/Time Field.
  2. Select a desired format in the Date/Time Format dialog.
  3. The system then updates final.dbf, filling other5 for each record using the chosen format (e.g., mm/dd/yy hh:mm → 10/10/25 14:45).
  4. If a record is missing either date or time, only the available part is written; missing components are left blank.

Notes

  • This process runs once per command; it is not automatic.
  • Existing other5 values are replaced during this operation.
  • Execution is batched for efficiency — performance impact is minimal even for large final.dbf files.

🕓 Auto Apply Date/Time

Purpose:
Enables automatic date/time stamping of new records as they are added to final.dbf, using the same formatting rules as Build Date/Time Field.

How It Works

  1. Enable Options → Auto Apply Date/Time (checkable menu item).
  2. When enabled for the first time, the Date/Time Format dialog appears so you can select your preferred date/time layout.
  3. On each refresh or incremental update:
    • Any record where other5 is empty is filled with the current local date/time in the chosen format.
    • Records that already contain a value in other5 are skipped.
  4. Disable at any time by unchecking Auto Apply Date/Time.

Example Output

10/10/25 14:45
10/10/25 15:22
10/10/25 16:08

Performance Notes

  • Only affects rows where other5 is blank.
  • Uses a single batch update rather than per-row writes, ensuring fast execution even as final.dbf grows throughout the day.

Together, these two features allow either manual or automatic control over how and when timestamps are applied to your data, enabling both end-of-day and continuous processing workflows.

  • Emergency Backups (United): Every time you click Refresh and UES detects a changed final file, it creates a United backup in the designated folder.
  • Restore Customer Files: UES creates a folder when a new customer is detected on open. If you accidentally switch customers mid-inventory and lose printed reports/audits, restore the previous store to continue.
  • MySQL Backups (MobTrans): Lists historical MySQL backups (not United backups). You can open versions to inspect, then Import New MySQL and select the known-good file to restore.

Quick Reference

Core actions

  • Refresh – sync UES with United changes
  • Quick Edit – bulk change/swap area/location IDs
  • Category Change – move partial/total dollars across categories
  • Manual Entry – rapid category/total input for a location
  • Split – separate merged locations by exact record (SKU/price)

When to use Details

  • Need multi-row edits with filters
  • Change location/category on many rows
  • Add/Delete records with live validation
  • MassEdit ranges by upload/area/location
