What is United Edit Suite?

United Edit Suite (UES) is a companion tool for the United inventory platform. It automatically detects the connected United version and loads the active customer’s data. The default view lists areas that have been counted and not marked as printed; items already printed disappear from this list, and items not yet counted don’t appear. Filters let you include printed items or show not-yet-counted areas.

Menu depth rule: Keep United open but do not be deeper than one level inside submenus while using UES. Being in the final file or >1 level deep can lock files and trigger errors until you back out. Use UES freely from high-level menus like Inventory Proofing.

Use the Refresh button whenever changes occur in United (new uploads, items marked as printed). Refresh syncs those changes into UES.

Selecting Areas

  • Click to select individual areas; click–drag for multi-select.
  • With areas selected, choose View Reports (financial-style) or View Scan Reports (pieces/units style).

Reports (Financial-style)

The View Reports window mirrors United’s printed report style and supports in-place editing with helpful tooltips.

Visibility controls

  • Toggle Categories and Category Groups to adjust report granularity.
  • When viewing by groups, clicking a group number toggles (hides/shows) underlying categories.

Bulk actions

  • Move area: Click an area number to move all its locations to a new area number.
  • Rename area: Click an area description to edit or annotate.
  • Relabel location: Click a location number to renumber. You can target one or multiple employees or All.
  • Edit location description: Click description to change text or add notes. Right-click → Mark as Gone prefixes “Gone ” to the name.
  • New Location…: Add a location with description/notes. This writes to United Location Setup and seeds a prior of $0 so the location appears on reports.
  • Quick Edit…: Quickly change or swap area/location numbers (e.g., swap 03090↔03100, set multiple to 03070).
  • Move category: Click a category number to move it. If grouped, pick a category with data.

Visual indicators

  • 🧑‍🤝‍🧑 Multiple employees counted this location.
  • ⚠️ Location total is $0 but there were priors.

    • Red category rows = previously counted here, but missing in current inventory.

Split Window

  1. Open Edit on a location, choose employee (if multiple).
  2. Pick the exact boundary record (by SKU or price point, e.g., $4.49). UES previews remaining totals for current vs. new location/area.
  3. Use Search Record if multiple matches exist; choose field (price, SKU, etc.), then select the unique match to complete the split.

Category change & manual entry (in-report edit)

  • Category Change…: Choose any category present in the location, pick a new category, and enter the amount to shift (e.g., move $500 from 14 → 3). UES applies the adjustment and updates totals.
  • Manual Entry…: Like United’s manual entry but prefilled with the chosen location. Enter category + total repeatedly for fast backroom adds; close to commit all entries.
  • Delete Location…: Delete entire location, only a specific upload in the location, or just one employee’s portion.

Zero-total locator

If a location shows $0 and you’re unsure where its counts went: click the location number. Because total is zero, UES opens a helper that lets you pick who counted it, shows all their locations with totals (including merged cases), and suggests five likely matches based on proximity. Double-click a candidate, verify in Details, then Update Location to move the data.

Details window

Details is similar to United’s final file view. By default it’s read-only; click Unlock to enable edits for supported fields.

Search & filter

  • Global search across category, price point, SKU, or description.
  • Right-click a cell to filter to that value (e.g., show all for a SKU).

Editing

  • Edit location and category on a single record.
  • Drag to multi-select rows, then Quick Change… to update all (or Delete).
  • Add Record creates a blank line (file name = Edit Suite).
  • When you edit a SKU, UES validates against the live customer master; price/description (and hidden fields like cost) update automatically.

MassEdit

  • Select a beginning record then click MassEdit. Fields auto-populate.
  • Clear a field to apply to all (no need to type “ALL”).
  • Leaving end record blank extends to the last record UES finds within the parameters (e.g., last record for upload 13 in area 40041, location 04-010).
  • Use Save Changes; if you try to exit with unsaved edits, UES prompts you to save.

View Details from Reports shows only the selected location. View Details (All) shows details across all currently viewed areas. In Scan Reports, View Details (Final File) opens the entire final file.

Scan Reports (Pieces/Units)

Identical functionality to Reports, but the layout is optimized for pieces/units style rather than financial three-variance format.

All Uploads

  • WinTakes-like view of every upload: who counted it, dollar and unit totals. Values reflect the final file (i.e., after adjustments).
  • Delete Upload to remove a selected upload.
  • View Locations within an upload, or open Details filtered to just that upload.

Audits (Desktop & Tablet)

Audits support sorting on all fields, show grand totals, and capture full change history.

Preparing & printing

  • Search by name, area, or location. Toggle options (e.g., hide employee names on printed audits).
  • Select audits → Print Audits to generate PDFs. UES remembers what’s been printed even after closing. Reprint specific audits by using Re-Enable.
  • Export Audits mirrors printing: exports only the enabled set.

Tablet workflow (same network)

  1. Click Get Address to obtain the device URL (IP + port) and open it on a tablet.
  2. On tablet, view audits by Name or by Location. Review includes: add record, mark for deletion, change SKU (invalid SKUs highlight red), edit quantity; for financial inventories, price and category can be edited too.
  3. When a location is finished, mark it Complete. Completed names/locations disable or disappear accordingly.

Importing & resolving audits

  1. On desktop, click Import Audits to pull tablet changes.
  2. Review the feed: added records (SKU/price + units), deletions, SKU changes (before/after, including price changes), unit changes, and not-found SKUs.
  3. Use Accept All, Reject individual items, or leave some pending to revisit later. Pending items remain visible until resolved.

Daily backups of audits (PDF + logs)

  • Use Back Up Exports at day’s end to select a folder. UES creates an /audits subfolder with a PDF per location (green = accepted, red = rejected, no color = unchanged).
  • Each import produces a text log capturing what changed, when the import occurred, and delta amounts per record.
Shortcuts: Accept All, Re-Enable (to reprint), and Enable All (auto-create audits for each import).

Options & Automation

  • Auto-update Areas: For each incoming location, if its current area differs from the most recent prior’s area, UES updates the current to match the prior (fixes forgotten area changes).
  • Auto-fix increments of locations: If a location ends with 1 or 6 and didn’t previously exist, UES adjusts it to end with 0 or 5.
  • View Production: Displays current production metrics for all employees.

Backups & Restore

  • Emergency Backups (United): Every time you click Refresh and UES detects a changed final file, it creates a United backup in the designated folder.
  • Restore Customer Files: UES creates a folder when a new customer is detected on open. If you accidentally switch customers mid-inventory and lose printed reports/audits, restore the previous store to continue.
  • MySQL Backups (MobTrans): Lists historical MySQL backups (not United backups). You can open versions to inspect, then Import New MySQL and select the known-good file to restore.

Troubleshooting

Symptom Cause Fix
Error banner after Refresh; can’t open Final File views United has a File locked (too deep in menus) Back out to a top-level menu (≤1 level deep) in United; then Refresh in UES
Expected changes not visible United data updated but UES not synced Click Refresh in UES
Location shows $0 with priors Counts moved/merged elsewhere Click location number → use Zero-total locator to find & move
Duplicate/odd location numbers (…1/…6) Increment typo during counting Enable Auto-fix increments in Options
Wrong area on a location Counter forgot to change area Enable Auto-update Areas; or Quick Edit/Move to correct
Tablet shows red SKU SKU not in customer master Correct the SKU; UES validates live and updates price/desc

Quick Reference

Core actions

  • Refresh – sync UES with United changes
  • Quick Edit – bulk change/swap area/location IDs
  • Category Change – move partial/total dollars across categories
  • Manual Entry – rapid category/total input for a location
  • Split – separate merged locations by exact record (SKU/price)

When to use Details

  • Need multi-row edits with filters
  • Change location/category on many rows
  • Add/Delete records with live validation
  • MassEdit ranges by upload/area/location
